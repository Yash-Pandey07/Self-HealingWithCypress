# Dashboard Data Workflow

This GitHub Actions workflow automatically publishes test results and self-healing statistics to the `dashboard-data` branch after each test run.

## What It Does

1. **Runs Cypress Tests** - Executes your self-healing test suite with Gemini API integration
2. **Collects Healing Stats** - Gathers test results and selector healing data into `healing-results.json`
3. **Processes Historical Data** - Aggregates results using `generate-historical-data.js`
4. **Publishes to Dashboard Branch** - Pushes `history.json` and `latest.json` to the `dashboard-data` branch

## Data Files

### `latest.json` (Full Run Payload)
Contains the complete data for the most recent test run:
- **Run Metadata**: `runId`, `runNumber`, `workflowName`, `repository`, `branch`, `commitSha`, `event`, `generatedAt`
- **Statistics**: `totalRuns`, `totalTestsCompleted`, `totalSelectorHeals`, `totalFlowHeals`, `totalFailures`, `estimatedTimeSaved`, `healSuccessRate`
- **Full Events Array**: `events` - all healing events from this run (the raw `healing-results.json` content)
- **Reference**: `file` - path to the run's dedicated JSON file in the `runs/` directory

## How It Works

1. **Branch Creation** (First Run)
   - If `dashboard-data` branch doesn't exist, the workflow creates it with an orphan commit
   - This is a clean, separate history from your main codebase

2. **Data Processing** (Every Run)
   - Fetches existing `history.json` from the `dashboard-data` branch
   - Combines with current run data
   - Maintains max 250 entries (oldest entries pruned automatically)
   - Generates both `latest.json` and updated `history.json`

3. **Publishing** (Every Run)
   - Commits changes to `dashboard-data` branch
   - Pushes to remote repository
   - Only commits if there are actual changes

## Setup Requirements

- GitHub Actions secrets configured:
  - `GEMINI_API_KEY` - For Gemini API calls in Cypress tests
- Repository permissions: `contents: write` (already configured)

## First Run

The workflow will:
1. Create the `dashboard-data` branch automatically
2. Generate initial `history.json` and `latest.json`
3. Push everything to the remote

No manual intervention needed!

## Troubleshooting

**Workflow fails on first run?**
- Check that `GEMINI_API_KEY` secret is configured
- Verify Actions have write permissions in repository settings

**Branch not showing up?**
- The workflow only commits when there are changes
- If tests produce no healing events, no commit is made
- This is expected behavior

**History not updating?**
- Check the Actions tab for workflow run logs
- Verify `healing-results.json` is being generated by Cypress
- Ensure `generate-historical-data.js` runs without errors

## Customization

To modify the history limit (currently 250):
- Edit `src/tools/generate-historical-data.js`
- Change line: `if (history.length > 250) history = history.slice(0, 250);`
- Update to desired maximum
